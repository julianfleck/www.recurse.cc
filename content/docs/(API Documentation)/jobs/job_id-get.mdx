Retrieve the status of any async job by its ID.

Track the progress of document processing jobs started via `POST /documents`.
Jobs run in the background using Celery workers and persist across service restarts.

**Job Statuses:**
- `PENDING`: Job queued but not started
- `PROCESSING`: Job actively running
- `COMPLETED`: Job finished successfully
- `FAILED`: Job encountered an error
- `RETRY`: Job failed but will retry
- `REVOKED`: Job was cancelled

**Job Types:**
- **Document Ingestion**: Full pipeline processing (parsing, frames, embeddings)
- **Section Processing**: Individual section analysis
- **Frame Extraction**: Semantic frame identification
- **Embedding Generation**: Vector embedding creation

**Response Format:**
```json
{
  "job_id": "ingest_ai_research_20240715_103000",
  "status": "PROCESSING",
  "message": "Processing section 3 of 10: Methodology",
  "progress": {
    "current": 3,
    "total": 10,
    "percentage": 30
  },
  "result": null,
  "created_at": "2024-01-15T10:30:00Z",
  "updated_at": "2024-01-15T10:32:15Z"
}
```

**Completed Job Result:**
```json
{
  "job_id": "ingest_ai_research_20240715_103000",
  "status": "COMPLETED",
  "message": "AI research paper processed successfully",
  "result": {
    "document_id": "ai_research_paper_2024",
    "sections_created": 8,
    "frames_extracted": 47,
    "embeddings_generated": 55,
    "processing_time": 127.5
  }
}
```

done
```
